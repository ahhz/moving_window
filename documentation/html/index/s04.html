Summary

The Moving Window library works with data that conforms to the Raster Concept. Geographical raster layers carry information of the spatial distribution of some variable. The data is stored in a matrix that corresponds to a regular grid and where the each cell, or pixel, corresponds to a square square geograpical area. To convert from matrix indices to geographical coordinates it is necessary to know the orientation of the grid, its corner coordinates and the size of each cell. The moving window library is not concerned with these, because it assumes that all raster layers that are used are identical in orientation, extent and cell size.

Thus a Raster is not more than a matrix. The operation on Raster that are supported are to query the dimension of the matrix and to access values. Specifically the moving window library needs to iterate over pixels and edges (pairs of adjacent pixels). For this the Raster most have associated views and iterators.
