namespace moving_window {

template<typename T> gdal_raster<T> open_gdal_raster(const boost::filesystem::path& path, GDALAccess access, int band = 1)

}

Description

This function opens a TIFF raster file dataset at the specified path and returns a gdal_raster<T> associated with the opened raster dataset. The user must specify the type of access as either either GAReadOnly or GAUpdate. Only a single band of the dataset is read, by default this is the first band.

Definition

The function is defined in "moving_window/utility.h"

Type requirements

T must be POD and fit within datatype of the raster dataset.

Preconditions

The file specified as path must exist and be a geotiff file.

Example of use

The following example opens an int raster for read and write access.

auto ras = moving_window::open_gdal_raster<int>("my_raster.tiff", GAUpdate);

}

Description

This function creates a TIFF raster file dataset at the specified path and returns a gdal_raster<T> associated with the opened raster dataset. There are two overloads of this function, either the user specifies the datatype for the raster dataset, or a default is selected based on the template argument T.

Definition

The function is defined in "moving_window/utility.h"

Type requirements

T must be POD and fit within the space associated. If the datatype is not specified native_data_type<T>::type is the default.

Preconditions

The file specified as Path must not exist, or if it exists may not be opened. Any existing file will be deleted and replaced by the new raster dataset

Example of use

The following example creates an integer raster of 50 rows by 75 columns.

auto ras = create_gdal_raster<int>("my_raster.tiff", 50,75);

}

Description

This function creates a TIFF raster file dataset at the specified path and returns a gdal_raster<T> associated with the opened raster dataset. The created dataset has the same number of rows and columns as the model dataset and also has the same spatial reference and cell size. It does not copy over the values of model.

Definition

The function is defined in "moving_window/utility.h"

Type requirements

T must be POD and fit within the space associated. If the datatype is not specified native_data_type<T>::type is the default.

Preconditions

The file specified as path must not exist, or if it exists may not be opened. Any existing file will be deleted and replaced by the new raster dataset

Example of use

The following example creates an integer raster that uses the same grid as the input file "model.tiff".

auto model = open_gdal_dataset<double>("model.tiff");
auto ras = create_gdal_raster_from_model<int>("my_raster.tiff", model);

square_window make_window(const square_tag&, int radius)

circular_window make_window(const circle_tag&, double radius)

}

Description

Creates a window object that can be used in the make_moving_window_view function. Depending on the overload either a circular or a square window is created. For square windows, the radius is the distance from side to the central pixel.

Definition

The function is defined in "moving_window/window.h"

The tags square_tag and circle_tag are defined in "moving_window/traits.h"

Example of use

The following example creates a window object that specifies a circular window with a radius of 5 pixels.

auto win = moving_window::make_window(moving_window::circle_tag(), 5);

namespace moving_window {

template<typename IndicatorTag, typename Window, typename Raster>

moving_window_view<IndicatorTag, Window, Raster>

make_moving_window_view(const IndicatorTag&, Window& window , Raster* raster)

template<typename IndicatorTag, typename Window, typename Raster , typename WeightRaster>

moving_window_view<IndicatorTag, Window, Raster, WeightRaster>

make_moving_window_view(const IndicatorTag&, Window& window , Raster* raster, WeightRaster* weight)

}

Description

This function creates a moving_window_view object which presents a view of the input raster (and optionally the weight raster) according to the indicator specified by IndicatorTag and the window object (that typically results from the make_window function.

Definition

The function is defined in "moving_window/moving_window_view.h".

Type requirements

Window must conform to the (not yet formalized) Window concept. IndicatorTag must be a tag associated with a (not yet formalized)IndicatorConcept class. Raster and WeightRaster must conform to the (not yet formalized) raster concept, for instance gdal_raster<int>

Preconditions

Raster and WeightRaster must exist and have the same size

Example of use

The following creates a moving_window_view.

auto ras = moving_window::open_gdal_raster<int>("input.txt");
auto win = moving_window::make_window(moving_window::square_tag(), 5);
auto tag = moving_window::area_weighted_patch_size_tag();
auto view = moving_window::make_moving_window_view(tag, win, ras);
